BEV Target Selection Validation Studio

A production-development BEV replay tool used to investigate commercial-vehicle target-selection false detections by running real logs through production TOS/ODP logic.

01

Context

During commercial-vehicle ADAS development, some target-selection problems appeared only in rare road and traffic conditions, especially around bus-only-lane or wide-road scenarios with surrounding vehicles. Reproducing those cases repeatedly with a real bus was slow and expensive.

The team’s original logic and signal flow existed in MATLAB/Simulink and workspace-based development scripts, so the validation tool had to fit that environment instead of replacing it.

02

Problem

When a false target selection occurred, the team needed to replay the same situation repeatedly, inspect how TOS/ODP logic behaved, and check whether parameter or logic changes reduced the false detection before spending more time on real-road confirmation.

03

Constraint

  • Rare bus-lane or large-road scenarios could not be reproduced on demand.
  • One real-road validation trip could take roughly 4–5 hours.
  • The analysis had to use the same production TOS/ODP logic path as the ECU-side development workflow.
  • Public portfolio content could not expose internal Simulink blocks, signal names, thresholds, or real logs.
04

Decision

I built a MATLAB/Simulink-based BEV validation studio that replays logged driving situations, injects ODP/TOS-related signals through MATLAB Base Workspace variables, visualizes target tracks and road geometry, and supports repeated logic tuning before final vehicle confirmation.

05

Impact

Production useUsed frequently during HMC commercial-vehicle production development across the commercial platforms listed on the HMC page.
Bus validationEspecially useful for bus-only-lane and wide-road scenarios that were hard to reproduce repeatedly.
Time reductionReduced repeated real-bus validation loops from about 10 trips to about 5 for similar issues, cutting 4–5 hour test trips roughly in half.
06

Implementation

  • MATLAB GUI for scenario replay and BEV visualization.
  • Simulink integration through MATLAB Base Workspace variables.
  • ODP/TOS signal mapping and update scripts.
  • Track-position, ego-path, road-radius, and target-selection visualization.
  • Replay workflow for false-detection debugging and parameter/logic review.
Real driving log ↓ MATLAB GUI replay controller ↓ Base Workspace signal injection ↓ Production TOS / ODP Simulink logic ↓ BEV target-selection review ↓ Reduced real-road re-test loop before final confirmation

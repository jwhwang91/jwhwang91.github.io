BEV Target Selection Validation Studio

MATLAB/Simulink replay and visualization tool for validating commercial-vehicle ADAS target-selection behavior.

MATLAB Script Simulink Base Workspace BEV Visualization TOS / ODP
01

Context

During commercial bus ADAS development, some target-selection issues appeared only in rare road and traffic situations, especially around bus-only-lane scenarios and dense surrounding vehicles.

The original development environment already depended on MATLAB/Simulink and MATLAB workspace variables. The validation tool therefore needed to fit the existing model workflow instead of replacing it with a separate application stack.

02

Problem

Reproducing the same scenario with a real bus after every logic or parameter change was expensive, slow, and often impractical. The team needed a way to repeatedly replay the issue, inspect TOS/ODP behavior, and validate modifications before final vehicle confirmation.

The core problem was not simply visualization. It was reducing the loop time between field issue, logic change, parameter tuning, and validation review.
03

Constraint

The original TOS/ODP logic and signal flow existed inside MATLAB/Simulink with workspace-driven scripts. The simulator had to integrate with that architecture and preserve the way engineers already exchanged variables between MATLAB scripts, the Base Workspace, and Simulink models.

  • Vehicle scenarios had to be replayed from logged driving data.
  • Signal replacement needed to work through MATLAB Base Workspace variables.
  • Simulink model integration had to support repeated ODP/TOS logic modification.
  • The output had to be understandable as a BEV target-selection debugging view.
04

Decision

I built a MATLAB/Simulink-based BEV simulator that replays logged driving situations, injects ODP/TOS-related signals through the MATLAB Base Workspace, visualizes target tracks and road geometry, and supports repeated logic tuning.

The architecture intentionally stayed close to the original development environment: MATLAB GUI for control and replay, Base Workspace for shared signal state, and Simulink models for ODP/TOS logic validation.

MATLAB GUI
├─ Load / replay driving log data
├─ Manage ODP/TOS signal mappings
└─ Draw BEV target and road geometry overlays

MATLAB Base Workspace
├─ Shared simulation variables
├─ Track position arrays and RoadRadius signals
└─ Input-replacement variables for Simulink

Simulink Model
├─ ODP obstacle data processing logic
├─ TOS target object selection logic
└─ To Workspace outputs for post-simulation review
05

Impact

The simulator reduced dependence on repeated real-bus testing for every parameter or logic change. It helped the team debug false target-selection behavior, tune logic more efficiently, and prepare changes before final real-vehicle confirmation.

Validation loopReplay the same edge case repeatedly instead of waiting for rare road conditions.
Engineering reviewInspect BEV target behavior and Simulink outputs before final vehicle confirmation.
06

Implementation

  • MATLAB GUI for scenario replay, signal loading, and BEV visualization.
  • Simulink model integration through Base Workspace variables.
  • ODP/TOS signal mapping and update scripts for simulation input replacement.
  • Track position, road-radius, and lane-related visualization overlays.
  • Replay-based debugging workflow for false target-selection behavior.
  • To Workspace output hooks for post-simulation review.
  • Support for repeated logic modification and parameter tuning before real-vehicle confirmation.
